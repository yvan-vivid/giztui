- [ ] **Step 1: Make `promptPreviewText` emit styled, escaped markup**

In `internal/tui/prompt_preview.go`, change `promptPreviewText` to bold/color the two headings and escape the user content (so brackets in templates aren't parsed as tview tags). Replace the function body's return with tview markup:

```go
func promptPreviewText(description, promptText string) string {
	desc := strings.TrimSpace(description)
	if desc == "" {
		desc = "(no description)"
	}
	tmpl := strings.TrimSpace(promptText)
	if tmpl == "" {
		tmpl = "(empty template)"
	}
	// [::b] bold headings; content is tview.Escape()d so template brackets render literally.
	return fmt.Sprintf("[::b]Description[::-]\n%s\n\n[::b]Template[::-]\n%s",
		tview.Escape(desc), tview.Escape(tmpl))
}
```

## Task 4: Stored prompt templates with literal `\n` → real newlines + repair migration

## Task 5: Render the applied-prompt result through the Markdown pipeline

**Files:** `internal/tui/prompts.go`, `internal/tui/bulk_prompts.go`

Root cause: prompt results are pushed to `a.aiSummaryView` (a TextView) via raw `SetText`, so Markdown tables render as raw pipes, `<br>` shows literally, and box-drawing becomes `??`. The reader pipeline (`render.MarkdownToTerminal(md, theme, width)`) is never applied. Render only the FINAL result (not per streaming token — instantiating glamour per token is expensive and streaming runs off the UI thread without QueueUpdateDraw).

- [ ] **Step 1: Add a small helper to render result text**

In `internal/tui/prompts.go` (or a shared spot like markdown.go), add:

```go
// renderPromptResult turns an LLM Markdown result into terminal-ready text using the same
// pipeline as the email reader (tables, headings, width-fit). <br> is normalized first
// because some models emit it inside table cells. Falls back to the raw text on error.
func (a *App) renderPromptResult(text string) string {
	clean := strings.ReplaceAll(text, "<br>", "\n")
	clean = strings.ReplaceAll(clean, "<br/>", "\n")
	clean = strings.ReplaceAll(clean, "<br />", "\n")
	out, err := render.MarkdownToTerminal(clean, a.Config.Rendering.GlamourTheme, a.getListWidth())
	if err != nil {
		return text
	}
	return out
}
```
Ensure imports `strings` and `github.com/ajramos/giztui/internal/render` are present in the file you put it in.

- [ ] **Step 2: Use it on the FINAL single-prompt result paths only**

In `internal/tui/prompts.go`, wrap the result text at the FINAL (non-streaming-token) `a.aiSummaryView.SetText(...)` sites with `a.renderPromptResult(...)`:
- the final streaming result (~line 563: `finalText := sanitizeForTerminal(result.ResultText)` → set `a.aiSummaryView.SetText(a.renderPromptResult(result.ResultText))`),

- the non-streaming fallback (~line 600).
Do NOT change the per-token streaming callback (~line 536) — leave it as the cheap incremental `sanitizeForTerminal` update; the final replacement renders properly once complete. Confirm the exact lines by reading the surrounding code; there should be exactly one streaming-token site to leave alone and the three final/complete sites to wrap.
